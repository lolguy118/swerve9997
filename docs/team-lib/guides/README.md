# Guides

Human-oriented tutorials and how-to references for Team271-Lib contributors
and robot project developers. Written for a first-season student; reviewers
and architects should start with [docs/planning/](../planning/README.md).

## Reading Order (New Contributors)

| Step | Document | Purpose |
| ---- | -------- | ------- |
| 1 | [start-here.md](start-here.md) | 5-minute orientation — mental models, 7 critical rules, guided reading path |
| 2 | [development-setup.md](development-setup.md) | Get Java 17, WPILib, and Gradle working; run tests locally |
| 3 | [docs/planning/README.md](../planning/README.md) | Navigate the planning hierarchy and find the right SDD |
| 4 | [input-shaping-guide.md](input-shaping-guide.md) | Understand joystick input curves before touching operator controls |
| 5 | [simulation-guide.md](simulation-guide.md) | Add physics simulation to a robot project using the library's sim layer |
| 6 | [sysid-workflow.md](sysid-workflow.md) | Characterize mechanisms using the library's SysID data capture tools |
